  <action>
Create `src/prayer.rs` with:

1. `PrayerResult` struct holding 6 `chrono::DateTime<chrono::Local>` fields: fajr, sunrise, dhuhr, asr, maghrib, isha.

2. `parse_method(s: &str) -> anyhow::Result<salah::prelude::Method>`: Match lowercase input string to Method enum variants. Accept these aliases:
   - "mwl" | "muslim_world_league" -> MuslimWorldLeague
   - "egyptian" -> Egyptian
   - "karachi" -> Karachi
   - "umm_al_qura" -> UmmAlQura
   - "dubai" -> Dubai
   - "moonsighting_committee" -> MoonsightingCommittee
   - "north_america" | "isna" -> NorthAmerica
   - "kuwait" -> Kuwait
   - "qatar" -> Qatar
   - "singapore" -> Singapore
   - "tehran" -> Tehran
   - "turkey" -> Turkey
   - _ -> bail with error listing all valid methods

3. `parse_madhab(s: &str) -> anyhow::Result<salah::prelude::Madhab>`: Match lowercase:
   - "shafi" | "shafii" | "standard" -> Shafi
   - "hanafi" -> Hanafi
   - _ -> bail with error

4. `calculate_prayers(lat: f64, lon: f64, method: Method, madhab: Madhab) -> anyhow::Result<PrayerResult>`:
   - Create `Coordinates::new(lat, lon)`
   - Get today's date with `chrono::Local::now().date_naive()`
   - Build configuration: `Configuration::with(method, madhab)` then set `.high_latitude_rule(HighLatitudeRule::recommended(coords))`
   - Build prayer schedule: `PrayerSchedule::new().on(date).for_location(coords).with_configuration(params).calculate()?`
   - Convert each prayer time from UTC to Local timezone with `.with_timezone(&chrono::Local)`
   - Return PrayerResult

5. `print_prayers(prayers: &PrayerResult, time_format: &str)`:
   - Determine format string: "12h" -> "%I:%M %p", else -> "%H:%M"
   - Print each prayer name (left-aligned, 10 chars wide) followed by formatted time
   - Order: Fajr, Sunrise, Dhuhr, Asr, Maghrib, Isha

6. Tests module with all tests from the behavior block above. For time-dependent tests (calculate_prayers), just assert the Result is Ok and the times are in chronological order -- do not assert exact times since they change daily.

Note on high-latitude test: The salah crate's `HighLatitudeRule::recommended(coords)` should handle Oslo. The test verifies no panic/error occurs. If the crate returns an error for extreme coordinates, wrap with anyhow context.
