  <name>Task 2: Create app.rs with App state, next_prayer, midnight rollover, Hijri/countdown formatting</name>
  <files>src/app.rs</files>
  <behavior>
    - App::new(prayers, time_format, lat, lon, method, madhab) creates app with View::Clock, running=true
    - App::next_prayer() returns Some(("Fajr", time)) when Fajr is next upcoming prayer
    - App::next_prayer() returns Some(("Fajr", tomorrow_fajr_time)) after all today's prayers have passed (midnight rollover)
    - App::format_countdown() returns "Maghrib in 1:23:45" format string when Maghrib is next
    - App::format_countdown() returns "Fajr in H:MM:SS" after Isha (using tomorrow's Fajr)
    - App::format_date_line() returns "DD MonthName YYYY AH · DD Mon YYYY" format
    - App::toggle_view() switches Clock to Schedule and back
    - App::tick() recalculates prayers when date changes (crosses midnight)
    - View enum has Clock and Schedule variants
  </behavior>
  <action>
Create `src/app.rs` with:

1. `pub enum View { Clock, Schedule }` -- the two display modes.

2. `pub struct App` with fields:
   - `pub running: bool` -- controls event loop exit
   - `pub view: View` -- current display mode
   - `pub prayers: PrayerResult` -- today's cached prayer times
   - `pub tomorrow_fajr: Option<chrono::DateTime<chrono::Local>>` -- cached tomorrow's Fajr for post-Isha countdown
   - `pub time_format: String` -- "24h" or "12h"
   - `pub prayer_date: chrono::NaiveDate` -- date prayers were calculated for (detect midnight crossing)
   - `lat: f64, lon: f64` -- for recalculation
   - `method_str: String, madhab_str: String` -- for recalculation

3. `impl App`:
   - `pub fn new(prayers: PrayerResult, time_format: String, lat: f64, lon: f64, method_str: String, madhab_str: String) -> Self` -- sets prayer_date to Local::now().date_naive(), computes tomorrow_fajr by calling calculate_prayers_for_date with tomorrow's date
   - `pub fn next_prayer(&self) -> Option<(&str, chrono::DateTime<chrono::Local>)>` -- iterates through [Fajr, Sunrise, Dhuhr, Asr, Maghrib, Isha], returns first where time > now. If none found (after Isha), returns ("Fajr", self.tomorrow_fajr) if available.
   - `pub fn format_countdown(&self) -> String` -- calls next_prayer(), computes duration, formats as "PrayerName in H:MM:SS". If no next prayer, returns empty string.
   - `pub fn format_date_line() -> String` -- uses `hijri_date::HijriDate::from_gr()` to get Hijri date, formats as "DD MonthName YYYY AH \u{00B7} DD Mon YYYY" (middle dot separator). Uses chrono::Local::now() for Gregorian part.
   - `pub fn toggle_view(&mut self)` -- flips Clock <-> Schedule
   - `pub fn tick(&mut self)` -- checks if Local::now().date_naive() != self.prayer_date. If date changed, recalculates prayers and tomorrow_fajr, updates prayer_date.
   - `pub fn prayer_list(&self) -> [(&str, chrono::DateTime<chrono::Local>); 6]` -- returns array of (name, time) tuples for schedule view.

4. Unit tests:
   - `test_next_prayer_returns_future_prayer`: construct App with known times, verify next_prayer returns the correct one
   - `test_toggle_view`: verify Clock -> Schedule -> Clock
   - `test_format_date_line`: verify output contains "AH" and middle dot
   - `test_countdown_format`: verify format matches "X in H:MM:SS" pattern

Note: For tests that need PrayerResult with specific times, create a helper function that builds a PrayerResult with times relative to now (e.g., fajr=now-5h, sunrise=now-3h, dhuhr=now+1h, etc.) so tests are deterministic.

Add `hijri_date = "0.5.1"` to Cargo.toml dependencies.
