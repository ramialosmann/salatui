<task type="auto">
  <name>Task 1: Create ui.rs rendering and tui.rs event loop</name>
  <files>src/ui.rs, src/tui.rs, Cargo.toml</files>
  <action>
1. Add `ratatui = "0.30.0"` to Cargo.toml dependencies. Do NOT add a separate crossterm dependency -- ratatui 0.30.0 re-exports crossterm via `ratatui::crossterm::*`.

2. Create `src/ui.rs` with all rendering functions:

   `pub fn draw(frame: &mut ratatui::Frame, app: &App)`:
   - Layout: vertical stack with `Constraint::Length(1)` for date line, `Constraint::Fill(1)` for main content, `Constraint::Length(1)` for countdown.
   - Calls draw_date_line, then either draw_clock or draw_schedule based on app.view, then draw_countdown.

   `fn draw_date_line(frame: &mut Frame, area: Rect, app: &App)`:
   - Renders `App::format_date_line()` as centered `Paragraph` in default style (white/monochrome).

   `fn draw_clock(frame: &mut Frame, area: Rect, app: &App)`:
   - Gets current time, formats as HH:MM (no seconds, per user decision).
   - Calculates total width: 5 digits (H, H, :, M, M) x DIGIT_WIDTH + 4 gaps (1-2 col spacing between digits). Total roughly 34 columns.
   - Centers horizontally and vertically within area.
   - Check if area is too small (width < 34 or height < 5): render a "Terminal too small" message instead.
   - For each character in the time string, look up digit_index, get DIGITS pattern, render to frame buffer directly using `frame.buffer_mut()`. For each `true` cell, set symbol to U+2588 (full block). For `false`, set to space. Use default Style (monochrome, per user decision).
   - No border box around clock (borderless mode per user decision).

   `fn draw_schedule(frame: &mut Frame, area: Rect, app: &App)`:
   - Title: "Today's Prayer Times" centered.
   - Horizontal rule line of dashes.
   - For each prayer in app.prayer_list(): format as "  PrayerName    HH:MM" (or 12h format based on app.time_format).
   - Use app.next_prayer() to determine which prayer is next -- mark it with U+25B6 (triangle arrow) prefix.
   - Past prayers (time <= now): render in `Style::default().fg(Color::DarkGray)` (dim).
   - Future prayers: default style.
   - Center the schedule block vertically within area using Layout with Fill constraints.
   - No keybinding hints on screen (per user decision).
   - Hijri/Gregorian date line and countdown timer remain visible (they are in separate layout chunks, not part of this function).

   `fn draw_countdown(frame: &mut Frame, area: Rect, app: &App)`:
   - Renders `app.format_countdown()` as centered `Paragraph`. Example: "Maghrib in 1:23:45".

3. Create `src/tui.rs` with terminal management and event loop:

   `pub fn run(mut app: App) -> anyhow::Result<()>`:
   - Call `ratatui::init()` to set up terminal (raw mode, alternate screen, panic hooks).
   - Set up tick rate: `Duration::from_millis(250)` (4 ticks/sec for smooth second transitions).
   - Event loop:
     ```
     loop {
         terminal.draw(|f| ui::draw(f, &app))?;
         let timeout = tick_rate.saturating_sub(last_tick.elapsed());
         if ratatui::crossterm::event::poll(timeout)? {
             if let Event::Key(key) = ratatui::crossterm::event::read()? {
                 // Only handle KeyPress events (not Release/Repeat)
                 if key.kind == KeyEventKind::Press {
                     match key.code {
                         KeyCode::Char('q') => app.running = false,
                         KeyCode::Char('s') => app.toggle_view(),
                         _ => {}
                     }
                 }
             }
         }
         if last_tick.elapsed() >= tick_rate {
             app.tick();
             last_tick = Instant::now();
         }
         if !app.running { break; }
     }
     ```
   - After loop: call `ratatui::restore()` to clean up terminal.
   - Use a defer pattern or ensure restore is called even on error (wrap in a closure or use scopeguard-style approach, or just call restore in the error path too).
   - Do NOT call `terminal.clear()` inside the loop (ratatui double-buffers, clearing causes flicker).
   - Do NOT use `ratatui::run()` convenience method (it blocks on read() with no tick support).
  </action>
